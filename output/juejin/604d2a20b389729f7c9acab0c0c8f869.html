『前端干货篇』：你不知道的Stylus

最近沉迷学习（其实大部分时间都在看英雄联盟S8总决赛），把这几天在学的Stylus做个总结。这篇文章对学习Stylus的新手来说应该会有帮助~ 为了让大家代码观赏性更好，这里的代码都以截图的方式呈现，用的IDE是VSCode。

介绍

Stylus是个非常优秀的CSS预编译器，算是前端工作流的新宠。写起来十分简洁！究竟有多简洁呢，请看图

左边是Stylus的写法，右边是原生CSS的写法
不需要写分号！！不需要写冒号！！甚至连花括号都可以省略！！！
只需要简单的缩进即可，对比手写原生CSS代码简直...

安装

你可以使用yarn来安装（推荐）
(不知道yarn的小伙伴可以看我之前写的关于yarn用法的文章~ juejin.im/post/5bbb1b… )

yarn global add stylus

也可以使用npm来安装

npm install stylus -g

安装好了后就可以使用stylus的命令来编译生成css文件

用法

命令

stylus基本编译命令

stylus xxx.styl -o xxx.css   //xxx是你创建的文件名  -o 的意思是-out 输出css文件
复制代码

例如：这是我的文件目录， 首先创建stylus文件，文件名可以随便取，我这里叫style，文件后缀必须以 .styl 结尾

文件里的内容编写方式类似于css写法，但可以省略冒号，分号，花括号，足够简洁 然后在该文件目录下打开控制台(终端)输入stylus命令 你会发现stylus自动帮我们在本目录下编译并生成好了style.css文件 然后打开这个style.css文件，会发现格式什么的都帮我们自动补齐了 是不是很方便~
值得注意的是，在我们编写stylus文件的时候，一定要注意缩进，因为stylus就是根据缩进来识别选择器层级和对应css样式规则的
比如 让我们再使用stylus编译一次 会发现生成的css文件乱七八糟，各种报错！ 所以在编写stylus文件的时候一定要注意缩进

有没有发现每修改一次stylus文件就要再输入一次stylus命令进行重新编译，这样对于我们开发者来说极大的不便。
所以stylus提供了解决的方法，就是在原有基础上加上-w

stylus -w xxx.styl -o xxx.css   //在原有的基础上加了-w  w的意思是watch，也就是监听
复制代码

这样就可以实时监听修改，并实时编译。

当我们修改stylus文件时(修改完记得保存 Ctrl+S ) 可以看到控制台实时帮我们重新编译了 然后我们再看看css文件 这样就达到了实时监控，这也是我们现代前端工作流中需要的，实时监控更新。

选择器

  • 缩进
    开始也说了，stylus的缩进有很重要的意义，stylus就是根据缩进来区别选择器和CSS样式，所以写完选择器后，下一行务必注意缩进
  • 逗号
    Stylus就和CSS一样，允许你使用逗号为多个选择器同时定义属性。 也可以省去逗号，一个选择器占一行写，可以达到同样的效果
  • 嵌套
    stylus支持嵌套语法，相当于后代选择器
  • 父级引用
    父级引用简单来说就是：与父级的选择器作拼接。语法是在嵌套的选择器前加一个&字符，我们直接看例子吧
